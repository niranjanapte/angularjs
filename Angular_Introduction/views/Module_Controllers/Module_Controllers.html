AngularJS Modules

AngularJS supports modular approach

AngularJS module defines an application.

  • Application Module − used to initialize an application with controller(s).

  • Controller Module − used to define the controller.

AngularJS modules are container for the different parts of your application such as controllers, services, filters, directives, etc.

AngularJS modules define application and its logical entities that helps you to divide your application. So you can keep several modules inside app. AngularJS allow you to declare a module using the angular.module() method.

A module can contain one or more controllers, services, filters and directives. Similarly an app can contains several modules.

Syntax

Application Module

var app = angular.module('yourApp', []);

AngularJS Controller

A Controller Module


AngularJS application mainly relies on controllers to control the flow of data in the application.

A controller is defined using ng-controller directive. A controller is a JavaScript object containing attributes/properties and functions.

Each controller accepts $scope as a parameter which refers to the application/module that controller is to control.

AngularJS controller used to:

  • To intialized state of the $scope object.
  • To add behavior to the $scope object.

Different ways to define controller in AngularJS

Syntax 1 - Define Controller

function CustomerController($scope) {

$scope.customer = { name: "Jimi Scott" };

}


Example

Syntax 2 - Define Controller with Module

var app = angular.module('yourApp', []);

app.controller('yourController', function($scope) {});


Example
